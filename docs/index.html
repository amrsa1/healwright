Self-Healing Tests for Playwright

Don't let broken selectors break your build. Healwright uses AI to understand your intent and fix locators at runtime.

$ npm install healwright 
test('login flow', async ({ page }) => {
  // Self-healing locator with fallback
  await page.heal.locator(
    '.submit-btn',
    'Submit button on form'
  ).click();
  
  // Or just describe it, no selector needed!
  await page.heal.fill(
    '', 
    'Email input field', 
    'user@example.com'
  );
});

How It Works

01

Try First

Your original locator runs with a quick timeout. If it works, we're doneâ€”zero overhead.

02

Collect & Analyze

On failure, Healwright gathers DOM candidates and sends them to AI along with your intent description.

03

Validate

AI returns ranked strategies (testid, role, label). Each is validated until one matches a single visible element.

04

Cache & Reuse

The working selector is cached locally. Subsequent runs skip the AI entirely.
